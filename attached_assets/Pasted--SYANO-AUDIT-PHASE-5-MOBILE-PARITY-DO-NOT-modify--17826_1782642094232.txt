# TASK 4 — MOBILE UX ANTI-PATTERNS
# ══════════════════════════════════
Scan artifacts/mobile/app/ and artifacts/mobile/src/

A) Touch target size violations (Apple HIG: 44pt minimum):
   Find interactive elements with explicit small sizes:
   - style={{ width: 20, height: 20 }} on a touchable element
   - style={{ padding: 2 }} on a button
   These are too small for reliable touch

B) Missing TouchableOpacity/Pressable feedback:
   Find <View onPress={...}> — Views do NOT provide touch feedback
   Should be <TouchableOpacity> or <Pressable>

C) Hardcoded localhost URLs (repeat check from Phase 2 focused on mobile):
   Any fetch('http://localhost...') in mobile code

D) Synchronous AsyncStorage access:
   AsyncStorage is always async — find:
   - AsyncStorage.getItem() called without await
   - AsyncStorage used in a non-async function without .then()

E) Missing loading states:
   Screen components that make API calls with no loading indicator
   (no ActivityIndicator or skeleton while data fetches)

F) Navigation stack issues (expo-router specific):
   - router.push() used where router.replace() should be
     (e.g. after login → should replace, not push, so back button
     doesn't return to login screen)
   - Screens missing proper header back button handling

For each finding:
---
TYPE: SMALL_TOUCH_TARGET / NO_FEEDBACK / LOCALHOST_URL /
      ASYNC_STORAGE_SYNC / MISSING_LOADING / WRONG_NAVIGATION
FILE: artifacts/mobile/app/screens/ProductCard.tsx
LINE: 45
CODE: <View style={{ width: 20 }} onPress={handleFav}>
ISSUE: View has no touch feedback, 20px too small (min 44pt)
FIX: <TouchableOpacity style={{ padding: 12 }} onPress={handleFav}>
PRIORITY: MEDIUM
---
