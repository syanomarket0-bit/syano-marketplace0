Current page is not sufficient for operational use.

Redesign the courier assignment experience.

For each order, display:

## Order Information

* Order ID
* Order date
* Order status

## Customer Information

* Customer name
* Customer phone
* Delivery zone
* Full delivery address

## Seller Information

* Seller name
* Store name
* Seller phone (if available)

## Order Summary

* Product count
* Product thumbnails
* Product names
* Total order value
* Delivery fee

## Courier Information

When selecting a courier display:

* Courier name
* Active assignments count
* Completed deliveries count
* Current status
* Capacity indicator

Available

Avoid assigning overloaded couriers.

Use modern cards.

No admin-looking tables.

# ISSUE 3 — Delivery Lifecycle Is Incomplete

Audit the entire workflow.

Verify the following transitions actually work:

ReadyForPickup
→ CourierAssigned

CourierAssigned
→ PickedUp

PickedUp
→ OutForDelivery

OutForDelivery
→ Delivered

OutForDelivery
→ DeliveryFailed

Currently there appears to be no completion stage after "Start Delivery".

Implement:

## Delivered

Button:

Mark Delivered

Transition:

OutForDelivery
→ Delivered

Required effects:

* Order status updated
* Customer notified
* Seller notified
* Admin notified
* Courier earnings recorded
* Courier statistics updated

## Delivery Failed

Button:

Report Delivery Failure

Transition:

OutForDelivery
→ DeliveryFailed

Allow failure reason selection:

* Customer unavailable
* Wrong address
* Customer rejected order
* Unable to reach customer
* Other

Store reason in database.

Notify seller and admin.
