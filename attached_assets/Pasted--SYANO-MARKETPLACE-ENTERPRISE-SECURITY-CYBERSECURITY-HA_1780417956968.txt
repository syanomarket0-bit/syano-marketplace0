# SYANO MARKETPLACE — ENTERPRISE SECURITY & CYBERSECURITY HARDENING AUDIT

You are performing a full production-grade security audit of the Syano marketplace.

Assume the application is preparing for public launch.

Assume hidden vulnerabilities exist until proven otherwise.

This is NOT a feature task.

This is NOT a UI task.

This is NOT a performance task.

Your objective is to identify, verify, document, and harden every realistic security weakness across the entire stack.

---

## MISSION OBJECTIVE

Achieve maximum practical security without breaking existing functionality.

Protect:

* Customers
* Sellers
* Admins
* Orders
* Messaging
* Notifications
* Authentication
* Uploaded content
* API endpoints
* Database

Maintain full compatibility with existing features.

PHASE 1
AUTHENTICATION AUDIT
--------------------

Audit:

POST /auth/register
POST /auth/login
POST /auth/logout
GET /auth/me

Verify:

* JWT generation
* JWT validation
* JWT expiration
* JWT secret handling
* Logout behavior
* Session invalidation logic

Check for:

* Token forgery
* Weak secret handling
* Replay attacks
* Session fixation
* Token leakage
* Missing expiration validation

Verify:

No endpoint trusts client-provided roles.

Server must derive role from validated JWT only.
