# SYANO — MARKETPLACE-GRADE SEARCH SUGGESTIONS ENGINE (AMAZON / NOON STYLE)

STOP.

The current search overlay behavior is wrong.

It currently behaves like a product finder.

This is NOT the desired experience.

We do NOT want product cards inside the search dropdown.

We do NOT want users seeing product results before entering the search page.

We want a true marketplace search suggestion system similar to:

* Amazon
* Noon
* AliExpress
* Trendyol
* Etsy

The dropdown should help users discover WHAT to search for, not immediately show products.

---

# GOAL

Transform the homepage search bar and global navbar search into a real Search Suggestion Engine.

The dropdown must show:

* Search intents
* Search phrases
* Related searches
* Category suggestions
* Store suggestions
* Trending searches

NOT product cards.

NOT product grids.

NOT product previews.

Products should only appear on:

* Shop page
* Search results page

after the search is executed.

---

# PART 1 — REMOVE PRODUCT RESULTS FROM DROPDOWN

Remove:

* Product cards
* Product images
* Product prices
* Product ratings
* Product previews

from the search dropdown.

Do NOT delete backend functionality.

Only remove them from the suggestion layer.

Search results page remains unchanged.

---

# PART 2 — BUILD SEARCH SUGGESTION ENGINE

Create a dedicated Search Suggestion Engine.

New endpoint:

```text
/api/search/suggestions
```

This endpoint must NOT return products.

It must return:

```ts
{
  suggestions: [],
  categories: [],
  stores: [],
  trending: []
}
```
