### BACKEND — New API Endpoint

**Endpoint:** GET /api/search/suggestions?q={query}&lang={ar|en}&limit={5}

**Response shape (strictly typed):**
```typescript
interface SearchSuggestion {
  type: 'intent' | 'category' | 'store' | 'trending';
  label: string;        // display text (Arabic or English based on lang param)
  labelAr: string;      // always include Arabic label
  labelEn: string;      // always include English label
  icon: string;         // icon identifier: 'search' | 'tag' | 'store' | 'fire'
  query: string;        // the search query to execute when user clicks
  meta?: string;        // optional: e.g. "42 products", "in Electronics"
}

interface SuggestionsResponse {
  suggestions: SearchSuggestion[];
  recentSearches: string[];   // from session/localStorage (handled frontend)
  trendingSearches: string[]; // top 5 from DB query_logs table
  processingTimeMs: number;
}
```

**Backend logic (implement ALL of these):**

1. INTENT SUGGESTIONS — reuse existing intent analysis from Step 3.
   - If query contains price intent (رخيص, cheap, بسعر, affordable) → add suggestion: "cheapest [detected category] results"
   - If query contains quality intent (أفضل, best, عالي الجودة) → add suggestion: "top rated [detected category]"
   - If query maps to a known category → add category suggestion

2. CATEGORY SUGGESTIONS — query the categories table:
   ```sql
   SELECT id, name_ar, name_en, product_count
   FROM categories
   WHERE name_ar ILIKE '%{processed_query}%' 
      OR name_en ILIKE '%{processed_query}%'
      OR fts_vector @@ to_tsquery('arabic', '{processed_query}')
   ORDER BY product_count DESC
   LIMIT 3
   ```

3. STORE SUGGESTIONS — query stores table:
   ```sql
   SELECT id, name, logo_url, verified, rating
   FROM stores
   WHERE name ILIKE '%{query}%'
      OR description ILIKE '%{query}%'
   AND is_active = true
   ORDER BY rating DESC, verified DESC
   LIMIT 2
   ```

4. TRENDING SUGGESTIONS — build a query_logs table if it doesn't exist:
   ```sql
   CREATE TABLE IF NOT EXISTS query_logs (
     id SERIAL PRIMARY KEY,
     query TEXT NOT NULL,
     lang VARCHAR(2) DEFAULT 'ar',
     result_count INTEGER,
     clicked BOOLEAN DEFAULT false,
     created_at TIMESTAMP DEFAULT NOW()
   );
   ```
   Then fetch top trending:
   ```sql
   SELECT query, COUNT(*) as search_count
   FROM query_logs
   WHERE created_at > NOW() - INTERVAL '7 days'
     AND query ILIKE '%{query}%'
   GROUP BY query
   ORDER BY search_count DESC
   LIMIT 3
   ```

5. LOG every search query to query_logs asynchronously (fire-and-forget, never block the response).

6. DEBOUNCE-READY: the endpoint must respond in under 100ms. Run all DB queries in parallel using Promise.all(). Never run them sequentially.

### FRONTEND — Navbar Search Dropdown

**Files to modify:** The existing Navbar component + SearchBar component (find them, don't create duplicates).

**Behavior:**
- User types 2+ characters → debounce 200ms → call /api/search/suggestions
- Show dropdown BELOW the search input with rounded corners and subtle shadow
- Dropdown disappears on: Escape key, click outside, clear input
- Keyboard navigation: Arrow Up/Down moves highlight, Enter executes search

**Dropdown sections (in order):**

```
[Recent Searches]          ← only if user has recent searches in localStorage
  🕐 موبايل سامسونج
  🕐 حذاء رياضي

[Suggestions]              ← from API
  🔍 موبايل سامسونج رخيص   (type: intent, sorted by price)
  🏷️ إلكترونيات            (type: category, 142 products)
  🏪 متجر التقنية           (type: store, ⭐ 4.8)

[Trending Now]             ← from API trendingSearches
  🔥 آيفون 15
  🔥 شوز نايك
```

**Component requirements:**
- Full RTL support (direction: rtl for Arabic, ltr for English)
- Each suggestion row: icon on the right (RTL) + label + meta text
- Store suggestions show store name + rating badge
- Category suggestions show category name + product count
- Intent suggestions show search phrase + intent badge (e.g. "الأرخص سعراً")
- Clicking any suggestion: sets input value + navigates to /shop?q={query}&intent={type}
- Arrow key navigation highlights rows with background change
- Loading state: show 3 skeleton rows while fetching
- Empty state: show "لا توجد اقتراحات" / "No suggestions found"

**Accessibility:**
- role="combobox" on input, role="listbox" on dropdown, role="option" on each row
- aria-expanded, aria-activedescendant properly set
- Keyboard trap handled correctly
