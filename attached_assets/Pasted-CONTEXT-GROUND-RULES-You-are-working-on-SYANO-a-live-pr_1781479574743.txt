── ENDPOINT 1: Overview ──────────────────────────────────
GET /api/admin/search-analytics/overview
Query params: none
Auth: admin role only (use existing admin auth middleware)

Response shape:
{
  period7: {
    totalSearches: number,
    zeroResultCount: number,
    zeroResultRate: number,      // percentage, 2 decimal places
    clickThroughRate: number,    // (clicked=true / total) * 100, 2 decimal places
    avgResultsCount: number      // 2 decimal places
  },
  period30: { ...same shape... },
  languageBreakdown: {
    arabic: number,              // count where lang = 'ar', last 30 days
    english: number,             // count where lang = 'en', last 30 days
    arabicPct: number,
    englishPct: number
  },
  processingTimeMs: number
}

DB logic:
- Use Drizzle ORM — no raw SQL strings unless Drizzle cannot express it
- Run all independent queries in parallel via Promise.all()
- Filter by created_at >= NOW() - INTERVAL '7 days' and '30 days'
- Zero result: result_count = 0 (verify actual column name in schema first)

── ENDPOINT 2: Top Queries ───────────────────────────────
GET /api/admin/search-analytics/top-queries
Query params (Zod-validated):
  - days: number, default 7, max 90
  - limit: number, default 20, max 100

Response shape:
{
  queries: Array<{
    query: string,
    count: number,
    zeroResults: boolean,        // true if avg result_count = 0
    avgResultsCount: number,
    clickThroughRate: number,    // percentage
    lang: string
  }>,
  processingTimeMs: number
}

DB logic:
- Group by query text, count occurrences
- Calculate avg result_count per query
- Calculate CTR per query (sum of clicked=true / total count)
- Order by count DESC
- Filter to last `days` days

── ENDPOINT 3: Zero-Result Queries ──────────────────────
GET /api/admin/search-analytics/zero-results
Query params (Zod-validated):
  - days: number, default 7, max 90
  - limit: number, default 20, max 100

Response shape:
{
  queries: Array<{
    query: string,
    count: number,
    lastSearched: string,        // ISO date of most recent occurrence
    lang: string
  }>,
  processingTimeMs: number
}

DB logic:
- Filter where result_count = 0
- Group by query, count
- Order by count DESC
- These represent inventory gaps — most searched = most urgent gap

── ENDPOINT 4: Daily Trends ──────────────────────────────
GET /api/admin/search-analytics/trends
Query params (Zod-validated):
  - days: number, default 30, max 90

Response shape:
{
  trends: Array<{
    date: string,                // 'YYYY-MM-DD'
    totalSearches: number,
    zeroResults: number,
    uniqueQueries: number
  }>,
  processingTimeMs: number
}

DB logic:
- Group by DATE(created_at)
- Count total, count where result_count = 0, count distinct query values
- Order by date ASC
- Last `days` days only
