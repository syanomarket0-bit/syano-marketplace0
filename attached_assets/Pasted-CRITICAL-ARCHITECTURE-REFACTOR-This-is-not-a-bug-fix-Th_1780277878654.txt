══════════════════════════════════════
NEW BUSINESS MODEL
══════════════════════════════════════

The marketplace must use a SINGLE ACCOUNT MODEL.

There will no longer be:

* Customer Account
* Seller Account

as separate account types.

Instead:

Every account starts as a Customer account.

A seller is simply a Customer account that has been approved for selling privileges.

══════════════════════════════════════
NEW USER JOURNEY
══════════════════════════════════════

Registration:

User
↓
Create Account
↓
Customer Account

There must be only ONE registration flow.

There must be only ONE login flow.

There must be only ONE account type at registration.

No seller registration.

No seller login.

No seller account creation.

══════════════════════════════════════
PERMISSIONS
══════════════════════════════════════

CUSTOMER:

* Browse
* Buy
* Wishlist
* Reviews

Cannot:

* Create products
* Manage store

══════════════════════════════════════

PENDING SELLER:

Same permissions as customer.

Additional:

* View application status

Cannot:

* Sell products
* Access seller dashboard

══════════════════════════════════════

SELLER:

Customer permissions +

* Create products
* Manage store
* View seller dashboard
* Process orders

══════════════════════════════════════

ADMIN:

Full permissions.

══════════════════════════════════════
OPEN YOUR STORE BUTTON
══════════════════════════════════════

This becomes the only seller onboarding entry point.

Behavior:

If user is not logged in:
→ Redirect to login.

If logged in as customer:
→ Open seller application.

If application pending:
→ Show application status.

If approved seller:
→ Open seller dashboard.
