CRITICAL ORDER WORKFLOW SYSTEM REDESIGN

This is one of the most important marketplace systems.

The goal is to build a professional order lifecycle similar to Amazon, Noon, Trendyol, Etsy and Shopify marketplaces.

Do NOT implement a simple status field.

Build a complete order workflow architecture.

══════════════════════════════════════
OBJECTIVE
══════════════════════════════════════

Create a structured order lifecycle with clear state transitions.

Every order must move through a controlled workflow.

No invalid transitions should be possible.

══════════════════════════════════════

══════════════════════════════════════

Before implementing:

Audit:

* Database schema
* Orders table
* Seller dashboard

Identify all areas affected by order status changes.

Provide implementation plan before coding.

══════════════════════════════════════
ORDER LIFECYCLE
══════════════════════════════════════

Implement the following workflow:

Pending
↓
Processing
↓
Shipped
↓
Delivered

The workflow must be strictly one-directional.

No backward transitions.

Examples:

Allowed:

Pending → Processing

Processing → Shipped

Shipped → Delivered

Not allowed:

Delivered → Shipped

Shipped → Pending

Processing → Pending

══════════════════════════════════════
ORDER CANCELLATION
══════════════════════════════════════

Customer may cancel only before shipment.

Allowed:

Pending → Cancelled

Processing → Cancelled

Not allowed:

Shipped → Cancelled

Delivered → Cancelled

After cancellation:

Seller loses permission to modify the order.

Cancelled orders become locked.

══════════════════════════════════════
SHIPPING STAGE
══════════════════════════════════════

When seller changes order status to:

Shipped

The system must require:

* Estimated delivery time
* Shipping company (optional)
* Tracking number (optional)

Estimated delivery time should be defined by the seller at shipment time.

Not before.

Not by the customer.

══════════════════════════════════════
SELLER PERMISSIONS
══════════════════════════════════════

Seller may:

* Accept orders
* Process orders
* Ship orders
* Mark delivered

Seller may NOT:

* Reverse statuses
* Edit cancelled orders
* Modify completed orders improperly
