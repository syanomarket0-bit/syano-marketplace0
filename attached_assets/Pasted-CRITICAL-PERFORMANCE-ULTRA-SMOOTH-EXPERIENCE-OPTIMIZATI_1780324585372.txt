CRITICAL PERFORMANCE & ULTRA-SMOOTH EXPERIENCE OPTIMIZATION

This task is currently the highest priority in the entire marketplace.

The primary objective is:

Make the marketplace feel instant, ultra-fast, fluid, and app-like across all devices.

HOWEVER:

Speed improvements must NEVER introduce bugs.

Speed improvements must NEVER break existing functionality.

Speed improvements must NEVER damage business logic.

Speed improvements must NEVER create UI regressions.

Stability is mandatory.

Performance improvements must be safe, measured, and production-grade.

══════════════════════════════════════
MISSION
══════════════════════════════════════

The marketplace should feel as close as possible to:

* Amazon
* Trendyol
* Noon
* Shopify storefronts

Users should feel that pages open instantly.

Navigation should feel smooth and effortless.

The goal is maximum possible speed and smoothness within the current architecture.

══════════════════════════════════════
CRITICAL SAFETY REQUIREMENTS
══════════════════════════════════════

Before changing anything:

1. Create a checkpoint / snapshot.
2. Audit the entire performance architecture.
3. Measure before optimization.
4. Only implement improvements that are proven beneficial.
5. Verify functionality after every optimization.

Do NOT perform risky refactors without validation.

Do NOT remove functionality.

Do NOT break:

* Authentication
* Seller system
* Customer system
* Orders
* Cart
* Checkout
* Products
* Search
* Notifications
* Dashboard
* Mobile navigation

After every optimization verify that behavior remains unchanged.
