CRITICAL PRODUCT SYSTEM & PERFORMANCE ARCHITECTURE UPGRADE

This is NOT a simple feature request.

The goal is to redesign the product system and image pipeline so the marketplace remains extremely fast, smooth, and scalable even with:

* Thousands of sellers
* Tens of thousands of products
* Hundreds of thousands of images
* Large traffic volumes
* Heavy mobile usage

Every decision must prioritize:

1. Performance
2. Scalability
3. Mobile experience
4. Long-term maintainability

Think like Amazon, Etsy, Shopify, Noon.

Do NOT implement quick fixes.

Do NOT implement basic upload functionality.

Build a production-grade architecture.

══════════════════════════════════════
PHASE 1 — FULL PERFORMANCE AUDIT
══════════════════════════════════════

Before implementing anything:

Audit the entire product system.

Investigate:

* Product creation flow
* Product listing flow
* Product page flow
* Image uploads
* Image delivery
* Search requests
* Homepage product loading
* Category loading
* Seller store loading
* API payload sizes
* Database queries
* React rendering performance
* Bundle sizes
* Mobile performance

Identify real bottlenecks.

Do not guess.

Provide findings before optimization.

══════════════════════════════════════
PHASE 2 — MULTI-IMAGE PRODUCT SYSTEM
══════════════════════════════════════

Upgrade product creation.

Allow sellers to upload:

* 1 cover image
* Up to 10 gallery images

Requirements:

* Mobile upload support
* Desktop upload support
* Drag and drop support
* Image preview
* Image reordering
* Cover image selection
* Image deletion
* Edit existing images

The seller experience must feel modern and professional.

══════════════════════════════════════
PHASE 3 — PROFESSIONAL IMAGE PIPELINE
══════════════════════════════════════

This is the most important part.

Never serve original uploaded images directly.

When a seller uploads an image:

Automatically generate:

1. Thumbnail version
2. Medium version
3. Large version

Example:

Thumbnail:
Used for:

* Homepage
* Search
* Categories
* Product cards
* Store pages

Medium:
Used for:

* Product details page

Large:
Used for:

* Gallery zoom
* Fullscreen viewing

Only load the size actually required.

Never load large images where small images are sufficient.

══════════════════════════════════════
PHASE 4 — IMAGE COMPRESSION
══════════════════════════════════════

Convert uploaded images automatically.

Preferred format hierarchy:

1. AVIF
2. WebP
3. JPG fallback

Goal:

Maximum visual quality.

Minimum file size.

Example:

A 3MB JPG should become a highly optimized image that loads dramatically faster while maintaining visual quality.

Avoid oversized PNG assets whenever possible.

══════════════════════════════════════
PHASE 5 — RESPONSIVE IMAGE DELIVERY
══════════════════════════════════════

Do not send desktop-sized images to mobile devices.

Serve image sizes dynamically based on:

* Device size
* Screen resolution
* Viewport requirements

Mobile users must receive lighter assets.

Desktop users may receive larger assets when necessary.

══════════════════════════════════════
PHASE 6 — LAZY LOADING
══════════════════════════════════════

Implement aggressive image lazy loading.

Rules:

Only visible images should load initially.

Images below the viewport should not load until needed.

Homepage:

Only above-the-fold images load immediately.

Everything else loads progressively.

Reduce initial network usage as much as possible.

══════════════════════════════════════
PHASE 11 — API OPTIMIZATION
══════════════════════════════════════

Reduce payload sizes aggressively.

Example:

Homepage product cards should NOT fetch:

* Full descriptions
* Full galleries
* Large metadata
* Unused fields

Homepage cards should only fetch:

* Product ID
* Title
* Thumbnail
* Price
* Rating
* Seller name

Additional data should load only when entering the product page.
