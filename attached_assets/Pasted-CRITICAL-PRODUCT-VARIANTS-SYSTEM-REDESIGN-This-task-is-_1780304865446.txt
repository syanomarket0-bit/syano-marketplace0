CRITICAL PRODUCT VARIANTS SYSTEM REDESIGN

This task is one of the most important architectural upgrades in the marketplace.

Do NOT implement a simple solution.

Do NOT add a few extra fields.

Build a true marketplace-grade variants system similar to Amazon, Shopify, Etsy, Noon and Trendyol.

══════════════════════════════════════
DATABASE ARCHITECTURE
══════════════════════════════════════

Design a scalable architecture.

Avoid storing variants as comma-separated text.

Avoid storing variants as unstructured JSON blobs if it causes future limitations.

Use a proper relational structure.

Example:

Product
↓
ProductVariant
↓
Variant Options

Each variant must have its own:

* ID
* SKU
* Stock quantity
* Price adjustment
* Status
* Image (optional)

The design must support future growth.
