══════════════════════════════════════
STRICT FORWARD-ONLY FLOW
══════════════════════════════════════

Order statuses must move ONLY forward.

Allowed:

Pending → Processing

Processing → Shipped

Shipped → Delivered

Not allowed:

Processing → Pending

Shipped → Processing

Delivered → Shipped

Delivered → Pending

Or any other backward transition.

Once a status is changed, it cannot be reversed.

CUSTOMER CANCELLATION RULE
══════════════════════════════════════

Exception:

The customer may cancel the order ONLY before shipment.

If the customer cancels:

* Order becomes Cancelled.
* Seller loses permission to modify that order.
* Seller can no longer change status.
* Seller can no longer continue the workflow.

Cancelled orders become read-only for the seller.
