CRITICAL SELLER ONBOARDING FIX

The platform now uses a SINGLE ACCOUNT MODEL.

There must NOT be any seller registration flow.

There must NOT be any seller account creation flow.

There must NOT be any seller login flow.

Every seller is an existing customer account that gains seller privileges after approval.

══════════════════════════════════════
PROBLEM
══════════════════════════════════════

Currently the "Open Your Store" button still redirects users to a seller registration/account creation flow.

This is incorrect and contradicts the new architecture.

══════════════════════════════════════
REQUIRED BEHAVIOR
══════════════════════════════════════

If user is NOT authenticated:

Open Your Store
→ Redirect to /login

After successful login:
→ Redirect to seller application page

══════════════════════════════════════

If user is authenticated as customer:

Open Your Store
→ Open seller application page directly

No account creation.

No registration.

No seller signup.

══════════════════════════════════════

If user has an existing pending application:

Open Your Store
→ Show application status page

Do not create a new application.

══════════════════════════════════════

If user is already an approved seller:

Open Your Store
→ Redirect to seller dashboard
