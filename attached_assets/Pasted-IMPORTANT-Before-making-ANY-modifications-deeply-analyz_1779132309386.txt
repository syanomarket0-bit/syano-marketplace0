SECTION 2 — CORE ORDER NOTIFICATION FLOW (HIGH PRIORITY)
==================================================

This is the MOST IMPORTANT workflow.

==================================================
A. CUSTOMER PLACES ORDER
==================================================

When a customer places an order:

The seller MUST instantly receive:
- a realtime notification
- notification badge update
- unread notification count update

The notification should include:
- customer name
- order number
- ordered products
- total price
- timestamp

Example:
"New order received from Ahmad — 3 products ordered."

==================================================
B. SELLER OPENS/SEES THE NOTIFICATION
==================================================

When the seller opens/views the notification:

Automatically:
- mark notification as read
- update unread count
- change order status from:
  "Pending"
  TO:
  "Processing"

IMPORTANT:
This transition must happen safely and consistently.

==================================================
C. SELLER ORDER WORKFLOW
==================================================

Implement clean notification-based order lifecycle updates.

Example flow:

1. Customer places order
→ Seller gets notification
→ Order status = Pending

2. Seller opens notification
→ Notification marked as read
→ Order status = Processing

3. Seller marks shipped
→ Customer gets notification

4. Seller marks delivered
→ Customer gets delivery notification

5. Customer confirms received
→ Final completion notification
