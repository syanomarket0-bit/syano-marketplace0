SYANO (سوق سوريا)

FLUID RESPONSIVE SCALING SYSTEM

CRITICAL REQUIREMENT

This project must NOT rely on traditional responsive design where elements stay the same size and only the layout changes.

Current problem:

As viewport width decreases:

* available space becomes smaller
* typography stays too large
* buttons stay too large
* cards stay too large
* spacing stays too large

Result:

The UI feels crowded.

Elements begin competing for space.

The layout eventually breaks visually.

---

DESIRED BEHAVIOR

The entire interface should scale proportionally as viewport size decreases.

Think of the experience as:

A controlled and intelligent zoom-out effect.

When screen width decreases:

Everything should gradually become smaller.

Not disappear.

Not hide.

Not collapse.

Scale.

---

GLOBAL SCALING TARGET

Scale proportionally:

Typography

Icons

Buttons

Cards

Images

Badges

Ratings

Prices

Inputs

Dropdowns

Navigation

Spacing

Padding

Margins

Gaps

Border Radius

Shadows

---

IMPORTANT

Do NOT solve this by hiding content.

Do NOT remove UI elements.

Do NOT reduce functionality.

Do NOT collapse sections unnecessarily.

The objective is scaling.

Not hiding.

---

TYPOGRAPHY

Replace rigid typography.

Use fluid typography.

Apply clamp() wherever appropriate.

Text should continuously scale between:

Desktop

Laptop

Tablet

Mobile

without abrupt jumps.

---

PRODUCT CARDS

This is the highest priority.

As viewport width decreases:

Product cards should gradually shrink.

Everything inside the card should shrink proportionally:

* image
* title
* rating
* review count
* price
* badges
* add button
* spacing

Cards should maintain identical visual proportions.

---

GRID SYSTEM

The grid should remain dense.

Avoid situations where:

screen width decreases

but card size remains fixed

causing overflow and clipping.

Cards should scale first.

Grid reflow should happen second.

---

SPACING SYSTEM

Spacing must scale.

Examples:

Large desktop spacing

↓

Medium laptop spacing

↓

Compact tablet spacing

↓

Mobile spacing

Maintain visual rhythm.

---

BUTTONS

Buttons must scale proportionally.

Avoid giant desktop buttons appearing on tablet screens.

---

NAVBAR

Scale:

* logo area
* search area
* navigation items
* action buttons
* settings controls

Avoid crowding.

---

VISUAL GOAL

The user should feel that the entire interface is intelligently resizing itself.

Not rearranging itself.

Not hiding itself.

Not breaking.

Scaling.
