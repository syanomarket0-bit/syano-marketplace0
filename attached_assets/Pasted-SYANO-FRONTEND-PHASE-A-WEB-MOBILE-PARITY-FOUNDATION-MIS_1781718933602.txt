SYANO (سوق سوريا)

FRONTEND PHASE A — WEB ↔ MOBILE PARITY FOUNDATION

MISSION

Your mission is NOT to redesign SYANO.

Your mission is to make the Mobile App and Marketplace Web App feel like the SAME PRODUCT.

The project already has a mature web frontend.

The web frontend is now the source of truth.

The mobile app must be audited and upgraded to achieve visual and structural parity with the web experience.

DO NOT redesign the web.

DO NOT introduce a new design language.

DO NOT invent new layouts.

Mirror the existing marketplace experience.

---

STRICT RULES

* Preserve all existing functionality.
* Preserve RTL support.
* Preserve Arabic and English.
* Preserve current SYANO branding.
* Preserve colors.
* Preserve spacing scale.
* Preserve typography hierarchy.
* Preserve shadows.
* Preserve border radius.
* Preserve animations where possible.
* No placeholder content.
* No mock data.
* No regressions.
* TypeScript = 0 errors.
* Expo compatibility must remain intact.

---

SOURCE OF TRUTH

Treat the Marketplace Web Frontend as the canonical implementation.

If Mobile and Web differ:

Web wins.

---

PHASE A OBJECTIVES

Create complete parity for:

1. Home Page
2. Global Navigation
3. Header
4. Footer / Bottom Navigation
5. Search Entry Experience

Do NOT continue to Product Pages, Cart, Checkout, Seller Areas, or Profile Areas.

Those belong to later phases.
