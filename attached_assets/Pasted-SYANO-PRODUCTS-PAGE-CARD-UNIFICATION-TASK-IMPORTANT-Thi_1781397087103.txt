SYANO (سوق سوريا) — PRODUCTS PAGE CARD UNIFICATION TASK

IMPORTANT:

This is NOT a redesign task.

This is NOT a new UI concept.

This is NOT an opportunity to create a different product card.

The Homepage Product Card is the official visual reference for the entire marketplace.

Your goal is to make Product Cards on the Products Page visually and structurally identical to the Product Cards used on the Homepage.

---

PROJECT CONTEXT

Brand:
SYANO
سوق سوريا

Current Homepage:
Homepage V7

Homepage Product Cards are already approved and represent the target marketplace quality.

The Products Page currently uses a different card implementation and must be aligned with the Homepage.

Do NOT change:

* Navbar
* Filters
* Search
* Categories
* RTL support
* Language system
* Currency system
* API integrations
* Data fetching logic

Only improve Product Card consistency.

---

PRIMARY OBJECTIVE

Refactor the Products Page so that it uses the same Product Card component, design system, styling rules, spacing, hierarchy, hover effects, badges, pricing layout, and visual language already used on the Homepage.

The Homepage Product Card must become the single source of truth.

Avoid duplicated card implementations.

---

REQUIRED IMPLEMENTATION

1. IDENTIFY CARD COMPONENTS

Locate:

* Homepage Product Card component
* Products Page Product Card component

Compare both implementations.

Determine whether:

* A shared ProductCard component already exists
  OR
* Multiple card implementations exist

---

2. CREATE A SINGLE SOURCE OF TRUTH

Preferred architecture:

<ProductCard />

or

<ProductCard variant="default" />

The Products Page should reuse the Homepage card rather than maintaining a separate visual implementation.

Remove UI divergence.

Do NOT duplicate styles.

---

3. MATCH HOMEPAGE VISUAL DESIGN EXACTLY

Products Page cards must match Homepage cards in:

* Card border radius
* Card background
* Card shadow
* Card hover animation
* Image presentation
* Image sizing
* Typography
* Price styling
* Badge styling
* Spacing
* Internal layout
* CTA button styling

No visual differences should remain unless absolutely required by functionality.

---

4. IMAGE SYSTEM

Use the same image behavior as Homepage cards:

* Same aspect ratio
* Same object-fit behavior
* Same image container height
* Same corner radius treatment
* Same hover interaction

All cards must appear visually consistent in the grid.

---

5. PRODUCT TITLE

Use Homepage title styling exactly:

* Same font size
* Same font weight
* Same line-height
* Same truncation behavior
* Same max lines

Long product names must not break grid consistency.

---

6. CATEGORY LABEL

Match Homepage implementation:

* Same position
* Same typography

* Same spacing

* Spacing

Discount display

* Colors

* Spacing

Homepage button design is the reference.

Products Page must use:

* Same button
