FIX 5 — File Upload Magic-Byte Validation
File: artifacts/api-server/src/routes/messaging.ts

Route: POST /conversations/:id/attachments
After receiving the file buffer, validate the actual file signature (magic bytes) before saving.
Add this validation:
typescriptconst ALLOWED_SIGNATURES: Record<string, number[][]> = {
  'image/jpeg': [[0xFF, 0xD8, 0xFF]],
  'image/png':  [[0x89, 0x50, 0x4E, 0x47]],
  'image/webp': [[0x52, 0x49, 0x46, 0x46]],
  'application/pdf': [[0x25, 0x50, 0x44, 0x46]],
}

function validateMagicBytes(buffer: Buffer, declaredMime: string): boolean {
  const signatures = ALLOWED_SIGNATURES[declaredMime]
  if (!signatures) return false
  return signatures.some(sig =>
    sig.every((byte, i) => buffer[i] === byte)
  )
}
Call this function after reading the file buffer. If validation fails, return 400 with message { error: 'invalid_file_type' }. Do not save the file.
Add the i18n key messaging.error.invalid_file_type to both ar.json and en.json:

Arabic: "نوع الملف غير مدعوم"
English: "File type not supported"
