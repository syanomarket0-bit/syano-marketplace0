Your job is to audit every page and component for missing error states, then fix all gaps. Do not redesign anything. Do not change any existing UI. Only add what is missing.

STEP 1 — AUDIT FIRST
Before writing any code, scan every file in artifacts/marketplace/src/pages/ and artifacts/marketplace/src/components/ and produce a list of every page/component that is missing any of these four states:

Loading — shown while data is fetching from API
Empty — shown when API returns success but data array is empty
Error — shown when API call fails or returns error status
Success — shown after user completes a mutation (create, update, delete)

Report the full list before proceeding.

STEP 2 — FIX ALL GAPS
For every missing state found in Step 1, add it using these exact rules:

Loading State Rule
Use the existing Spinner or Skeleton component already in the codebase.

Find it first — do not create a new one.
Pattern:
tsxif (isLoading) return <Spinner />
Or for page-level loading with layout:
tsxif (isLoading) return (
  <div className="flex items-center justify-center min-h-[400px]">
    <Spinner />
  </div>
)

Empty State Rule
Every empty state must have:

An icon (use existing icon set — do not import new libraries)
A translated title using i18n
A translated subtitle using i18n
A call-to-action button where relevant

Pattern:
tsxif (data.length === 0) return (
  <div className="flex flex-col items-center justify-center min-h-[400px] gap-4 text-center px-4">
    <Icon className="w-16 h-16 text-muted-foreground" />
    <h3 className="text-lg font-semibold">{t('page.empty_title')}</h3>
    <p className="text-sm text-muted-foreground">{t('page.empty_subtitle')}</p>
    <Button onClick={...}>{t('page.empty_action')}</Button>
  </div>
)

Error State Rule
Every error state must have:

A translated error message using i18n
A retry button that re-triggers the query

Pattern:
tsxif (isError) return (
  <div className="flex flex-col items-center justify-center min-h-[400px] gap-4 text-center px-4">
    <AlertCircle className="w-16 h-16 text-destructive" />
    <h3 className="text-lg font-semibold">{t('common.error_title')}</h3>
    <p className="text-sm text-muted-foreground">{t('common.error_subtitle')}</p>
    <Button variant="outline" onClick={() => refetch()}>{t('common.retry')}</Button>
  </div>
)

Success State Rule
Every mutation (form submit, delete, update) must show feedback.

Use the existing toast/notification system already in the codebase.

Find it first — do not create a new one.
Pattern:
tsxonSuccess: () => {
  toast({ title: t('page.success_message'), variant: 'success' })
}
onError: () => {
  toast({ title: t('common.error_title'), variant: 'destructive' })
}
