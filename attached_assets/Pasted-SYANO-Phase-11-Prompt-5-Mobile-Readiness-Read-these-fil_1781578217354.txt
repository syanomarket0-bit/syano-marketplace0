STEP 1 — AUDIT FIRST
Before writing any code, scan every file in artifacts/marketplace/src/pages/ and artifacts/marketplace/src/components/ and produce a list of every issue found in these categories:
A — Touch Targets Under 44px
Any interactive element (button, link, icon button, checkbox, radio, select) whose width or height is explicitly set below 44px.
B — Overflow on 375px
Any element with:

Fixed pixel widths wider than 375px
min-width values that exceed viewport on mobile
Horizontal scroll caused by content wider than viewport
Tables without horizontal scroll wrapper
Inline styles with fixed widths

C — RTL Layout Issues
Any element with:

Hardcoded left-* or right-* Tailwind classes instead of start-* or end-*
Hardcoded ml-* or mr-* instead of ms-* or me-*
Hardcoded pl-* or pr-* instead of ps-* or pe-*
text-left or text-right instead of text-start or text-end
Icons that should flip in RTL but don't have rtl:scale-x-[-1]
Flex rows that don't reverse in RTL

D — Modals and Dropdowns Breaking on Small Screens
Any modal, dialog, sheet, or dropdown that:

Has a fixed width wider than 90vw
Has no max-height with scroll for long content
Overflows the viewport on 375px
Has close button unreachable on small screens

E — Missing Mobile Navigation
Any page that:

Has no back button or way to navigate back on mobile
Has desktop-only navigation with no mobile equivalent

Report the full list with file paths and line numbers before proceeding.

STEP 2 — FIX ALL ISSUES
Touch Target Fix Rule
Every interactive element must be at least 44×44px.
For icon buttons:
tsx// Before
<button className="p-1">
  <Icon className="w-4 h-4" />
</button>

// After
<button className="p-2 min-w-[44px] min-h-[44px] flex items-center justify-center">
  <Icon className="w-5 h-5" />
</button>

Overflow Fix Rule
For tables on mobile, wrap in horizontal scroll:
tsx<div className="overflow-x-auto w-full">
  <table>...</table>
</div>
For fixed widths, replace with responsive equivalents:
tsx// Before
<div style={{ width: "600px" }}>

// After
<div className="w-full max-w-2xl">
For min-width values that overflow:
tsx// Before
style={{ minWidth: "22rem" }}

// After
className="w-full sm:min-w-[22rem]"

RTL Fix Rule
Replace all directional Tailwind classes:
WrongCorrectml-*ms-*mr-*me-*pl-*ps-*pr-*pe-*left-*start-*right-*end-*text-lefttext-starttext-righttext-endrounded-l-*rounded-s-*rounded-r-*rounded-e-*border-l-*border-s-*border-r-*border-e-*
For arrows and directional icons that must flip in RTL:
tsx<ChevronRight className="w-4 h-4 rtl:rotate-180" />

Modal Fix Rule
Every modal and dialog must:

Have max-w-[90vw] or w-[90vw] on mobile
Have max-h-[85vh] overflow-y-auto for long content
Have close button reachable at top of modal

Pattern:
tsx<DialogContent className="w-[90vw] max-w-lg max-h-[85vh] overflow-y-auto">

Dropdown Fix Rule
Every dropdown must:

Not exceed viewport width
Use w-full on mobile, fixed width on desktop:

tsxclassName="w-full sm:w-[280px]"
