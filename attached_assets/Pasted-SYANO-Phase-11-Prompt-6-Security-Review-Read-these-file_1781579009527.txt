STEP 1 — AUDIT FIRST
Before writing any code, scan every file in artifacts/api-server/src/routes/ and artifacts/api-server/src/middleware/ and produce a list of every security issue found in these categories:
A — Unauthenticated Routes
Every route that:

Modifies data (POST, PUT, PATCH, DELETE) without requireAuth
Returns sensitive user data without requireAuth
Should be protected but has no auth middleware

B — Wrong Role Access
Every route that:

Is under /admin/ path but missing requireRole('admin')
Is under /seller/ path but missing requireRole('seller')
Is under /courier/ path but missing requireRole('courier')
Allows a seller to access another seller's data
Allows a customer to access another customer's data

C — Missing Input Validation
Every route that:

Accepts user input without Zod validation
Accepts file uploads without type/size validation
Has no length limits on string inputs
Accepts numeric IDs without validating they are positive integers

D — Data Exposure
Every endpoint that returns:

Password hashes
Raw tokens or secrets
Other users' private data (email, phone, address) to non-admin users
Internal system fields not needed by the client

E — Rate Limiting Gaps
Check if rate limiting exists on:

POST /api/auth/login
POST /api/auth/register
POST /api/auth/forgot-password
Any endpoint that sends emails or SMS
Search endpoint (prevent scraping)

F — ID Ownership Validation
Every route that accepts a resource ID (:id, :orderId, :productId) and check:

Does it verify the resource belongs to the authenticated user?
Can a seller access another seller's orders by guessing an ID?
Can a customer access another customer's orders by guessing an ID?

Report the full list with file paths and line numbers before proceeding.

STEP 2 — FIX ALL ISSUES
Auth Middleware Fix Rule
Find the existing requireAuth and requireRole middleware — do not create new ones.
Pattern:
typescriptrouter.patch('/orders/:id/status',
  requireAuth,
  requireRole('seller'),
  async (req, res) => { ... }
)

Input Validation Fix Rule
Use Zod for all input validation. Find existing Zod schemas in the codebase and follow the same pattern.
For string inputs:
typescriptconst schema = z.object({
  name: z.string().min(1).max(200),
  description: z.string().max(2000).optional(),
  price: z.number().positive(),
  id: z.coerce.number().int().positive()
})
For file uploads — validate size and type:
typescriptconst MAX_FILE_SIZE = 5 * 1024 * 1024 // 5MB
if (file.size > MAX_FILE_SIZE) {
  return res.status(400).json({ error: 'file_too_large' })
}

Ownership Validation Fix Rule
For every route that accepts a resource ID, add an ownership check:
typescript// Orders — customer can only access own orders
const order = await db.query.orders.findFirst({
  where: and(
    eq(orders.id, orderId),
    eq(orders.customerId, req.user.id) // ownership check
  )
})
if (!order) return res.status(404).json({ error: 'not_found' })
typescript// Products — seller can only modify own products
const product = await db.query.products.findFirst({
  where: and(
    eq(products.id, productId),
    eq(products.sellerId, req.user.sellerId) // ownership check
  )
})
if (!product) return res.status(404).json({ error: 'not_found' })

Data Exposure Fix Rule
Never return these fields in any response:
typescript// Strip sensitive fields before returning
const { password, resetToken, verificationToken, ...safeUser } = user
return res.json(safeUser)
If using Drizzle select, explicitly select only needed columns:
typescriptconst user = await db.select({
  id: users.id,
  name: users.name,
  email: users.email,
  role: users.role
}).from(users).where(eq(users.id, id))

Rate Limiting Fix Rule
Check if express-rate-limit is already installed in package.json.
If yes — add rate limiters to auth endpoints:
typescriptimport rateLimit from 'express-rate-limit'

const authLimiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 minutes
  max: 10, // max 10 attempts
  message: { error: 'too_many_attempts' },
  standardHeaders: true,
  legacyHeaders: false
})

router.post('/auth/login', authLimiter, handler)
router.post('/auth/register', authLimiter, handler)
router.post('/auth/forgot-password', authLimiter, handler)
If express-rate-limit is NOT installed — do not install it. Instead, add a comment:
typescript// TODO: Add rate limiting — install express-rate-limit
And report it as a finding without implementing it.
