Migration SystemDual-layer approach:

Startup migrations (artifacts/api-server/src/lib/run-migrations.ts) — runs automatically every time the API server starts. Contains ADD COLUMN IF NOT EXISTS and CREATE TABLE IF NOT EXISTS guards. Handles additive changes only. Always idempotent.

Drizzle Kit (lib/db/drizzle.config.ts) — for full schema synchronization in development:

pnpm --filter @workspace/db run push

⚠️ Requires an interactive TTY (cannot be run headlessly). For production, use the startup migration pattern or manual SQL.

Required Setup Steps (fresh database)

1. Set DATABASE_URL to your PostgreSQL connection string

2. Apply full schema (dev only — requires TTY)

pnpm --filter @workspace/db run push

3. Seed the admin account

pnpm --filter @workspace/scripts run seed

4. (Optional) Load demo data

Import data.sql or syano_backup.sql.gz if migrating an existing instance

Creating an Admin Accountpnpm --filter @workspace/scripts run seed

This is idempotent — safe to run multiple times. It creates or updates the admin account defined in scripts/src/seed-admin.ts.

To change admin credentials before running in a new environment, edit the constants in scripts/src/seed-admin.ts:

const ADMIN_EMAIL    = "your-admin@email.com";const ADMIN_PASSWORD = "your-secure-password";const ADMIN_NAME     = "Admin";

Build & DeploymentStep 1: Clone / Fork the ProjectIn Replit: Fork the Repl to the new account.Outside Replit: Export the Repl as a zip, or use git if a remote is configured.

Step 2: Install Dependenciespnpm install

This installs all workspace packages. Requires pnpm v9+ and Node.js 24.

Step 3: Configure SecretsIn Replit: Tools → Secrets (not .env files — Replit injects secrets as environment variables).

Minimum required secrets for the app to start:

DATABASE_URL      = SESSION_SECRET    = <random 64-byte base64 string>

Optional secrets (enable features):

RESEND_API_KEY            = FROM_EMAIL                = noreply@yourdomain.comTWILIO_ACCOUNT_SID        = TWILIO_AUTH_TOKEN         = TWILIO_FROM_PHONE         = +1234567890TWILIO_VERIFY_SERVICE_SID = VAPID_PUBLIC_KEY          = VAPID_PRIVATE_KEY         = VAPID_EMAIL               = mailto@yourdomain.comCORS_ORIGIN               = https://yourdomain.comVITE_SUPPORT_PHONE        = +963XXXXXXXXX

Step 4: Provision the DatabaseOption A — Replit PostgreSQL (recommended):

In Replit: Tools → Database → Create databaseDATABASE_URL is set automaticallyOption B — External PostgreSQL:

Create a PostgreSQL 14+ instance (Neon, Supabase, Railway, etc.)Set DATABASE_URL secret manuallyStep 5: Run Schema Migrations

Apply full schema from Drizzle definitions (requires interactive TTY)

pnpm --filter @workspace/db run push

OR: Import from the SQL backup directly

psql $DATABASE_URL < schema.sqlpsql $DATABASE_URL < data.sql     # optional demo data

Step 6: Seed Admin AccountEdit scripts/src/seed-admin.ts to set your admin email/password, then:

pnpm --filter @workspace/scripts run seed

Step 7: Generate VAPID Keys (if enabling push notifications)npx web-push generate-vapid-keys

Copy the output into VAPID_PUBLIC_KEY and VAPID_PRIVATE_KEY secrets

Step 8: Start DevelopmentRestart each workflow in Replit's workflow panel, or:

Terminal 1 — API server

pnpm --filter @workspace/api-server run dev

Terminal 2 — Web marketplace

PORT= BASE_PATH=/ pnpm --filter @workspace/marketplace run dev
