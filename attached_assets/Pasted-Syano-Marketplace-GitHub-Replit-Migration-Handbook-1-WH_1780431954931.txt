4. DATABASE MIGRATION
Provider: Replit's built-in PostgreSQL 16 (provisioned per-account)
Connection: Standard pg connection pool via DATABASE_URL env var
ORM: Drizzle ORM (drizzle-orm/node-postgres)
Schema location: lib/db/src/schema/ (TypeScript source)
Migrations: No SQL migration files — Drizzle uses push (schema-diff and apply directly)
Schema applied: Automatically on every deployment via scripts/post-merge.sh → pnpm --filter db push

Can the existing database be reused?
A) Reuse existing database — YES, if you have the connection string.
The database is not stored in the GitHub repo, but its connection string is in the Replit secret DATABASE_URL. If you have that connection string, paste it as a secret in the new account and all existing data (users, products, orders) is immediately available.

B) New database (fresh start):

In the new Replit account, open Tools → Database
Click Create Database — Replit provisions PostgreSQL 16 and auto-populates DATABASE_URL as a secret
Start the API server workflow — runMigrations() in artifacts/api-server/src/index.ts applies the schema automatically on first boot
Verify in the Replit Database panel that tables were created
C) Full data migration (existing data → new database):

# On source machine with old DATABASE_URL:
pg_dump "$OLD_DATABASE_URL" --no-owner --no-acl -F c -f syano_backup.dump
# On new machine with new DATABASE_URL:
pg_restore "$NEW_DATABASE_URL" --no-owner --no-acl -d syano_backup.dump

Then set DATABASE_URL to the new connection string in Replit Secrets.

9. DEVELOPMENT CONTINUATION GUIDE
For the next developer picking this up:

Architecture to Know First
API contract: All routes defined in lib/api-spec/src/ as OpenAPI. Any new endpoint requires updating the spec, then running codegen. Do not write fetch calls directly — use the generated hooks from lib/api-client-react/.
DB changes: Never use drizzle-kit push with a TTY prompt interactively. Use executeSql (the code_execution sandbox) for additive changes (ADD COLUMN IF NOT EXISTS). Schema lives in lib/db/src/schema/.
i18n: Translation keys live in artifacts/marketplace/src/i18n/{en,ar}.json. Never hardcode display strings. Never do lang === "ar" ? "Arabic text" : "English text" inline.
Styling: Tailwind v4. Design tokens in artifacts/marketplace/src/index.css. No tailwind.config.js. Responsive tokens use clamp() CSS custom properties.
Priority Areas for Continued Work
Mobile (medium priority)

Expo mobile app (artifacts/mobile/) has feature parity gaps vs the web app — messaging, reviews, and seller store pages are not yet in the mobile app
Mobile RTL layout needs continued validation on physical devices
Image picker for product photos in mobile seller tools not yet implemented
Currency / Payments (high priority)

SYP ↔ USD conversion uses a static rate — needs a live FX API integration
Payment gateway not integrated — checkout creates orders but does not process payment
Push Notifications (low priority)

Web push works; mobile push (Expo expo-notifications) is not yet wired up
Notification preferences (per-user opt-in/opt-out per category) are UI-only, not persisted
SEO (low priority)

Sitemap is statically generated at build time — should be server-rendered or incrementally updated as products are added
No structured data (JSON-LD) on product pages yet
Security (ongoing)

File uploads (product images) validate type client-side only — server-side MIME validation should be added
Rate limiting on auth endpoints (/api/auth/login, /api/auth/register) exists but thresholds may need tuning for production traffic
Performance (low priority)

Admin analytics page loads all data on mount — could benefit from date-range pagination
seller/messages page does polling — could be upgraded to WebSocket or SSE
10. FINAL MIGRATION RISK REPORT
Risk	Probability	Impact	Fix
DATABASE_URL secret not set	High	Fatal — API crashes on boot	Always the first secret to add; Replit Database panel auto-sets it
SESSION_SECRET not set	High	All users get "dev-secret-change-in-prod" as signing key — insecure, tokens cross-compatible	Add a strong random value: openssl rand -hex 32
Empty database — no schema tables	High if new DB	All API calls 500 with "relation does not exist"	Start API server once — runMigrations() applies schema automatically on boot
VAPID keys change	Medium	All existing push subscriptions invalid, users stop receiving notifications	Re-subscribe all users (they'll be prompted on next visit); or port original VAPID keys from .replit
node_modules cache mismatch	Low	Native binaries compiled for wrong platform	pnpm install --frozen-lockfile (Nix ensures linux-x64 target is consistent)
.replit port mappings	Low	Preview pane shows wrong artifact	Replit re-discovers ports; artifact artifact.toml files define preview paths
Orval-generated client out of sync	Low	TypeScript errors if spec was hand-edited without regenerating	Run pnpm --filter @workspace/api-spec run codegen once after import
Email/SMS OTP not configured	Medium	Registration OTP only appears in server logs (dev fallback)	Add RESEND_API_KEY and Twilio secrets; or accept console-log OTP for dev
Custom domain DNS	Low	Old domain still resolves to old Replit account	Update DNS A/CNAME to new .replit.app deployment URL
Expo mobile packager domain	Medium	Mobile app can't connect to API in new repl	EXPO_PUBLIC_DOMAIN and REPLIT_DEV_DOMAIN are auto-injected — no action needed
pnpm-lock.yaml frozen mismatch	Very Low	Install fails if lock file doesn't match manifest	pnpm install (without --frozen-lockfile) to regenerate, then commit
Supply-chain minimumReleaseAge	Very Low	New packages blocked for 24h after publish	Use minimumReleaseAgeExclude in pnpm-workspace.yaml for trusted packages only
Summary: Minimum steps for a working import

1. Import from GitHub
2. Add DATABASE_URL secret   ← provision new DB or paste existing
3. Add SESSION_SECRET secret  ← openssl rand -hex 32
4. Start "API Server" workflow
5. Start "marketplace: web" workflow
6. Open preview — app is live

Everything else (VAPID, Twilio, Resend) is optional and degrades gracefully. The app is fully functional for browsing, auth, cart, checkout, and orders with only those two secrets set.
