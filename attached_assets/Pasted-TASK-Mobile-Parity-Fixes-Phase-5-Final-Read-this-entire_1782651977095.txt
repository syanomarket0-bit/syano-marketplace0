  - Where the locale is initialized on app start

  If present, it can be used for reloadAsync after forceRTL.

STEP 4: In the root _layout.tsx, add RTL enforcement on startup.
  Add BEFORE the root layout component renders:

  import { I18nManager } from "react-native";
  // Import your locale getter — check existing import pattern
  import { getLocale } from "../src/i18n";  // adjust path if needed

  // Add inside the root layout component, in a useEffect that runs once:
  useEffect(() => {
    const isArabic = getLocale() === "ar";
    if (I18nManager.isRTL !== isArabic) {
      I18nManager.forceRTL(isArabic);
      // Reload is required for forceRTL to take effect
      // Only reload if expo-updates is available
      // If not available, the change takes effect on next cold start
      try {
        const { reloadAsync } = require("expo-updates");
        reloadAsync();
      } catch {
        // expo-updates not available — RTL takes effect on next restart
      }
    }
  }, []);

STEP 5: IMPORTANT — verify the useEffect dependency array:
  This must run only ONCE on mount (empty deps []).
  The isRTL check prevents unnecessary reloads if RTL is already correct.

STEP 6: Check if locale can change at runtime (user switches language in settings):
  grep -rn "setLocale\|changeLanguage\|switchLanguage" artifacts/mobile/src/ --include="*.ts" --include="*.tsx"

  and trigger a reload when it changes.
