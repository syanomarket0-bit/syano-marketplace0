TASK 1 — FIX XSS: store.website UNSAFE href (HIGH — MOST CRITICAL)
════════════════════════════════════════
This is a stored XSS vulnerability. Fix it in TWO places: render + write time.

── PART A: Frontend render guard ──
FILE: artifacts/marketplace/src/pages/store/[slug].tsx

BACKGROUND:
Line ~676 renders seller-supplied store.website directly in an href:
  <a href={store.website} target="_blank" rel="noopener noreferrer">
A seller can set website to "javascript:alert(document.cookie)".
rel="noopener noreferrer" prevents tab-napping but does NOT block javascript: protocol.

STEP 1: Open the file and read it fully.
STEP 2: Add this utility function at the TOP of the file,
  before the component function, after all imports:

  function isSafeUrl(url: string | null | undefined): boolean {
    if (!url) return false;
    try {
      const parsed = new URL(url);
      return parsed.protocol === "https:" || parsed.protocol === "http:";
    } catch {
      return false;
    }
  }

STEP 3: Find the <a href={store.website}> element (line ~676).
  Replace:
    {store.website && (
      <a href={store.website} target="_blank" rel="noopener noreferrer">
        {store.website}
      </a>
    )}
  With:
    {isSafeUrl(store.website) && (
      <a href={store.website!} target="_blank" rel="noopener noreferrer">
        {store.website}
      </a>
    )}

  NOTE: The non-null assertion (!) is safe here because isSafeUrl already
  returns false for null/undefined — TypeScript needs the hint.

STEP 4: Search for any other place in this file that renders store.website in an href:
  grep -n "store\.website\|href.*website" artifacts/marketplace/src/pages/store/[slug].tsx
  Apply isSafeUrl guard to every href that uses seller-supplied URL data.

STEP 5: Run tsc --noEmit -p artifacts/marketplace/tsconfig.json → 0 errors.
