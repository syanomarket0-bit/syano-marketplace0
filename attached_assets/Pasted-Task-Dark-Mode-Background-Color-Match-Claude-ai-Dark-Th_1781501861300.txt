════════════════════════════════════════════════════════════════════
PHASE 2 — IMPLEMENT THE CHANGE
════════════════════════════════════════════════════════════════════

Based on Phase 1 findings, apply the change using the MINIMUM number
of edits — ideally ONE change in ONE place that propagates everywhere.

CASE A — If background is controlled via CSS custom property
(e.g. --background in .dark block in index.css):
  Change ONLY the --background value (and --card, --popover if they
  also use pure black) in the .dark block to the chosen target color.
  Do not touch any component files.
  Example:
    .dark {
      --background: 24 24 27;   /* zinc-900 in HSL space if using HSL */
      --card: 28 28 32;         /* slightly lighter than background */
    }
  Match the format already used (HSL space values, or hex, or RGB —
  whatever format the existing variables use).

CASE B — If background is a Tailwind color extension in config:
  Change ONLY the dark mode background entry in tailwind.config.ts
  to the chosen target color. Do not touch component files.

CASE C — If background is hardcoded via dark:bg-* classes scattered
across components:
  This is the worst case. Report this finding clearly. Then:
  FIRST: introduce a CSS variable --background in the .dark block
  in index.css with the chosen target color.
  SECOND: in tailwind.config.ts, map bg-background to
  `hsl(var(--background))` (or the appropriate format).
  THIRD: replace hardcoded dark:bg-black / dark:bg-zinc-950 classes
  with bg-background in ONLY the top-level layout wrappers
  (App.tsx, Layout.tsx, the outermost page wrapper) — do NOT
  refactor every single component, only the root-level containers
  that set the page background.

IMPORTANT RULES FOR THE CHANGE:
  - Change ONLY the main page background and card/surface backgrounds.
  - Do NOT change: text colors, border colors, button colors, accent
    colors, the primary green color, icon colors, or any color that
    is not a background surface.
  - The sidebar, navbar, cards, dropdowns, modals, and sheets should
    all become slightly lighter than the main background — they should
    NOT all become the same flat color. If they currently use
    dark:bg-zinc-950 for background and dark:bg-zinc-900 for cards,
    shift BOTH values by the same amount toward the target, maintaining
    the same relative lightness difference between them.
  - Do NOT change anything in the light theme.
  - Do NOT change the mobile Expo app theme in this task — web only.
