IMPROVEMENT 1 — Smart Empty State
──────────────────────────────────────────────────────────────────

When a search query returns 0 results, replace whatever currently
renders with a full empty state component that includes ALL of:

  1a. Clear visual indicator — use an existing icon from the icon
      library already used in this project (e.g. a search or
      magnifying glass icon with an X, or similar). Do NOT add a
      new icon library. Size: large enough to be seen immediately
      (e.g. w-16 h-16 or equivalent in existing scale).

  1b. Message — two lines:
      Primary: "لا توجد نتائج لـ "{query}""  (Arabic)
               "No results for "{query}""     (English)
      Secondary: "جرّب كلمة مختلفة أو تصفّح الفئات"  (Arabic)
                 "Try a different word or browse categories" (English)
      All strings must go into ar.json and en.json under key path:
      search.empty.title, search.empty.subtitle
      — follow the existing i18n key naming convention exactly.

  1c. "Clear Search" button — clicking it clears the query param
      from the URL and returns to /shop with no query. Use an
      existing Button component and existing variant (outline or
      secondary — whichever is used for secondary actions on other
      pages). Label: "مسح البحث" / "Clear Search" — add to i18n
      under search.empty.clearButton.

  1d. Trending / Popular searches fallback — below the button, show
      a row of suggestion chips pulled from the REAL query_logs
      table (queries with the highest result_count, last 30 days,
      limit 5). Label: "الأكثر بحثاً" / "Trending Searches".
      Add to i18n under search.empty.trending.
      CRITICAL: if query_logs has fewer than 3 entries with
      result_count > 0, do NOT show this section at all — never
      show fake or hardcoded suggestions. The existing
      /api/search/suggestions endpoint already returns trending
      data — use it rather than a new endpoint if it covers this.
      If a new endpoint is needed, add it following the exact
      pattern of existing search endpoints (Zod validation,
      try/catch, processingTimeMs, admin-safe).

  1e. "تصفّح الفئات" link — a secondary link below the chips that
      navigates to /categories. Use existing Link component.
      Label: "تصفّح جميع الفئات" / "Browse All Categories" —
      add to i18n under search.empty.browseCategories.

  Design rules for the empty state component:
    - Center the content vertically and horizontally in the results
      area (not full-page center — centered within the product grid
      column only, so the filter sidebar remains visible on desktop).
    - Match the spacing, font sizes, and color tokens used in other
      empty states in this codebase (check seller dashboard, order
      history, or wishlist for existing empty state patterns and
      REUSE the same structure/classes exactly).
    - RTL: all text right-aligned in Arabic, left-aligned in English.
    - Dark mode: all elements must use existing dark mode tokens.
    - Mobile 375px: same component, stacked correctly, no overflow.

IMPROVEMENT 2 — Keyword Highlighting in Product Card Titles
──────────────────────────────────────────────────────────────────

When a search query is active (/shop?q=...), highlight the matching
portion of the search query within the product card title text.

  2a. Implementation approach:
      Create a small pure utility function in a new file:
      artifacts/marketplace/src/utils/highlightMatch.ts
      The function signature: highlightMatch(text: string, query: string)
      Returns: Array<{part: string, highlight: boolean}>
      Splits text on case-insensitive, diacritic-insensitive matches
      of the query terms.

      For Arabic normalization, copy ONLY these two functions from
      searchProcessor.ts — rename them with a "frontend" prefix to
      avoid confusion (e.g. frontendNormalizeArabic,
      frontendRemoveTashkeel):
        - normalizeArabic()   → normalizes alef variants (أإآا → ا)
        - removeTashkeel()    → strips diacritics
      Do NOT import any other function from searchProcessor.ts.
      Do NOT import the server-side module into the frontend bundle.

      For English: case-insensitive match only, no normalization needed.

      Each {highlight: true} segment renders as:
        <mark className="bg-primary/20 text-primary rounded-sm px-0.5">
          {part}
        </mark>
      Use the existing `primary` color token — do not introduce a
      new highlight color.

  2b. Apply highlighting to:
      - Product card TITLE only (not description, not category, not
        store name — title only to keep it clean).
      - Only when debouncedQuery is non-empty.
      - Find the exact component file rendering the product title on
        the search results page and replace the plain text render
        with a .map() over the segments array — React elements only,
        no dangerouslySetInnerHTML.

  2c. Do NOT highlight in:
      - Autocomplete dropdown suggestions (already have their own styling)
      - Category names or store names on the card
      - Any page other than /shop search results
