IMPROVEMENT 1 — New Multi-Signal Ranking Formula
──────────────────────────────────────────────────────────────────

Replace the current formula with a new weighted multi-signal score.
Use ONLY signals confirmed as available in AUDIT B.

The new formula must include ALL of these signals (skip any signal
whose column does not exist in the DB — do not create new columns):

  Signal 1 — Text Relevance (weight: 40%)
    Keep the existing FTS + trigram text match score as-is.
    Do not change how text matching works — only its weight changes.

  Signal 2 — Quality Score (weight: 25%)
    Combine: avg_rating (normalized 0–1 by dividing by 5) and
    review_count (log-normalized: log(1 + review_count) / log(1 + max_count))
    Formula: quality = 0.6 * normalized_rating + 0.4 * log_review_count
    This prevents products with 1 review at 5 stars from beating
    products with 500 reviews at 4.8 stars.

  Signal 3 — Popularity Score (weight: 20%)
    Use order_count and/or view_count if available.
    Log-normalize each: log(1 + value) / log(1 + max_value)
    If both exist: popularity = 0.6 * norm_orders + 0.4 * norm_views
    If only one exists: use it alone.
    Log normalization prevents a product with 10,000 views from
    completely dominating one with 1,000 views.

  Signal 4 — Freshness Score (weight: 10%)
    Based on created_at. Products newer than 30 days get full score.
    Products older than 180 days get 0. Linear decay between.
    Formula: freshness = max(0, 1 - (age_in_days - 30) / 150)
    This gives new products a fighting chance without permanently
    suppressing older good products.

  Signal 5 — Availability Penalty (weight: 5%)
    If stock column exists:
      stock = 0     → multiply final score by 0.1 (bury out-of-stock)
      stock = 1–5   → multiply final score by 0.8 (low stock warning)
      stock > 5     → no penalty (multiply by 1.0)
    If stock column does not exist: skip this signal entirely.

  Signal 6 — Featured Boost (additive, not weighted)
    If is_featured or is_promoted column exists:
      Add 0.15 to the final normalized score (capped at 1.0).
    This ensures featured products appear prominently without
    completely overriding relevance.

  Signal 7 — Seller Trust Boost (additive, not weighted)
    If seller trust score or verification status column exists:
      Verified seller → add 0.05 to final score (capped at 1.0).
    If column does not exist: skip entirely.

  Final combined score:
    base_score = (0.40 * text) + (0.25 * quality) + (0.20 * popularity)
                 + (0.10 * freshness)
    after_penalty = base_score * availability_multiplier
    final_score = min(1.0, after_penalty + featured_boost + trust_boost)

  CRITICAL: The weights above (40/25/20/10/5) must sum correctly
  across the weighted signals. Do the math before implementing.
  If a signal is unavailable, redistribute its weight proportionally
  across the remaining signals — document the redistribution in a
  comment in the code.

──────────────────────────────────────────────────────────────────
IMPROVEMENT 2 — Seller Diversity (Anti-Flood)
──────────────────────────────────────────────────────────────────

  After ranking, apply a post-processing diversity step:
  No single seller may appear more than 3 times in the first 20
  results. If a seller has more than 3 products in the top 20,
  push the excess products down (after position 20) and fill their
  slots with the next highest-scoring products from other sellers.
  This prevents one seller from flooding the top results.
  Implement this in JS after the DB query returns results, not in SQL.
  Apply only when total results > 20. If results ≤ 20, no diversity
  step needed.

──────────────────────────────────────────────────────────────────
IMPROVEMENT 3 — New Products Discovery Boost
──────────────────────────────────────────────────────────────────

  For products created in the last 7 days with 0 orders and 0 reviews:
  Apply a one-time "new arrival" boost: add 0.10 to their base score
  (before availability penalty and featured boost).
  This is separate from Signal 4 (Freshness) — it specifically helps
  brand-new listings get initial visibility.
  Cap: this boost only applies to the first 500 products created
  per seller (prevent abuse by mass-listing).
  Implement as a flag in the query result or JS post-processing —
  whichever is cleaner given the existing architecture.
