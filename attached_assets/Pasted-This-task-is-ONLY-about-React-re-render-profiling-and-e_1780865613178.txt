This task is ONLY about React re-render profiling and elimination.

Do NOT work on:

* bundle size
* TTI/LCP
* scroll optimization
* virtualization
* memory optimization
* CPU profiling
* animations
* CSS effects
* networking
* API caching
* service worker
* image optimization
* lazy loading

Those categories have their own dedicated passes.

Your ONLY objective is to eliminate unnecessary React renders across the entire platform while preserving identical behavior.

IMPORTANT:

• Audit first.
• Measure first.
• Never optimize blindly.
• Never add React.memo/useMemo/useCallback everywhere.
• Only optimize PROVEN render bottlenecks.
• Verify every fix before moving to the next one.
• Do NOT continue to the next phase until the current phase is fully completed and verified.

==================================================
PHASE 1 — COMPLETE RENDER AUDIT
===============================

Perform a complete render analysis across:

• Marketplace (web)
• Admin Dashboard
• Seller Dashboard
• Customer Dashboard
• Mobile App (Expo)

Inspect every major screen:

* Home
* Products
* Product Detail
* Cart
* Checkout
* Orders
* Notifications
* Conversations
* Chat
* Seller Dashboard
* Admin Dashboard
* User Profile
* Settings

Profile actual render behavior.

Identify:

* Components rerendering without prop changes
* Context updates cascading through unrelated trees
* Parent renders recreating child props
* Arrays recreated every render
* Objects recreated every render
* Inline callbacks recreated every render
* Inline styles recreated every render
* Derived values recalculated every render
* Expensive map/filter/reduce operations
* React Query updates causing excessive rerenders
* Translation hooks causing tree-wide updates
* Search state propagating unnecessarily
* Form state rerendering entire pages
* Modal open/close rerendering unrelated components

Do NOT implement before the audit is complete.

==================================================
PHASE 2 — CONTEXT PROVIDER ANALYSIS
===================================

Audit ALL providers.

Including but not limited to:

AuthContext
CurrencyContext
GuestCartContext
NotificationProvider
Language/i18n provider
QueryClientProvider wrappers
Theme providers
Seller providers
Admin providers
Any custom context

Verify:

* value object stability
* callback stability
* derived values
* memoization correctness

Find cases like:

const value = {
user,
logout,
login,
settings
}

being recreated every render.

Apply useMemo ONLY where proven necessary.

Do NOT wrap unstable dependencies that still recreate every render.

==================================================
PHASE 4 — CALLBACK STABILITY
============================

Audit every:

onClick
onChange
onSubmit
onRemove
onDelete
onAddToCart
onFollow
onSearch
onFilter

Find inline callbacks:

onClick={() => ...}

Determine whether they break memoization.

Apply useCallback ONLY when:

* callback is passed to memoized children
* callback identity causes rerenders

Do NOT wrap local event handlers that never leave the component.

==================================================
PHASE 5 — ARRAY / OBJECT STABILITY
==================================

Find objects and arrays recreated every render.

Examples:

const links = [...]
const tabs = [...]
const sortOptions = {...}
const filters = [...]
const columns = [...]

Move to:

useMemo(...)

ONLY if identity affects child rerenders.

Do NOT memo static constants that can simply live outside the component.

Prefer module-level constants whenever possible.

==================================================
PHASE 6 — EXPENSIVE COMPUTATIONS

sort()

cart calculations
price calculations
notification counts
seller statistics
dashboard summaries

Find expensive computations executed every render.

Cache using useMemo ONLY if:

* computation is measurable
